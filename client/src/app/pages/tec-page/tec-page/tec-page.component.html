How It Works

Leveraging User Computing Power

Every time a user connects, they contribute not only consumption but also computing power. Our architecture leverages a P2P system to utilize users' computing power to sustain the network. To ensure trust and security, we employ cryptographic proof for data encryption and communication.

Network Structure

Tree Graph Structure

The network consists of users organized into branches of a tree graph. The first 63,000 users are connected directly to the server, forming the root of the tree graph. New users are assigned to a branch and inserted into the tree graph, allowing the app to balance itself dynamically. Each node in the connection graph is composed of 3 to 6 users connected via WebSockets.

Memory Model

Our memory model uses a B+-tree structure to create a multi-level virtual memory system:

  • Users can contact each other to get pointers to other users.
  • The tree uses public addresses as keys and IPs as values.
  • Depth of the tree depends on the number of tables, balancing latency and memory consumption.
  • Users have a cache for frequently used addresses and IPs, using predictive algorithms to prefetch data.
  • A redundancy protocol ensures data consistency and redundancy within each node.

Transaction Process

Transactions in our network are secure and efficient, thanks to our cryptographic proof system:

  • A user fetches another user or multiple users, connects, and generates a proof of authorization.
  • The user sends a request and proof to the users possessing the required data.
  • The receiving users verify the proof, perform the requested operation, and send back the updated proof and data.
  • The transaction completes once all users update their memory and close the connection.

Everything operates based on relative addresses from the memory model, ensuring smooth and secure transactions.
